# Adapter Family 11 — File / Spreadsheet Import Adapter *(A16)*

*Registry + defaults apply.*

**Purpose.** Ingest spreadsheets/CSVs/exports as **Observations + Evidence**, with **discovered, unconfirmed semantics**. **Not.** ### **NOT canonical truth — a spreadsheet is an external operational system with UNKNOWN semantics until mapped; a cell value is NEVER silently promoted to canonical truth.** **Direction.** bidirectional (read; write-back is a `CONSEQUENTIAL_EFFECT`).

**Identity.** file identity = `(tenant, source, content_digest)`; version identity = digest; ### **tab + row identity = a declared key (a header column or a composite) — NOT row position (rows reorder).** **Schema discovery.** ### **column meaning is DISCOVERED and confirmed (a mapping Claim), not assumed; a `manual_override` (a hand-edited cell) is detected vs a formula result; merged cells / duplicate rows / stale exports are flagged.** **Provenance.** ### **an imported value is `MODEL_EXTRACTED` (read off the sheet) at best — a consequential use requires deterministic confirmation or human assertion; `MODEL_INFERRED` interpretation never gates.** **Hostile #18:** ### **a spreadsheet row changes AFTER it was used in a proposal ⇒ at the checkpoint the row's value is re-read (a `CONSEQUENTIAL_FRESHNESS_READ` re-reads the current sheet version) ⇒ drift ⇒ the approval voids; the stale proposal never executes.** **Ops.** `import_file` (`OBSERVATION_ONLY`); `read_row` (read class per caller); `write_back` (`CONSEQUENTIAL_EFFECT`, READBACK_VERIFIABLE). **Reconciliation.** discrepancies vs canonical ⇒ `ConflictRaised`. **Acceptance.** `test_a16_cell_not_silently_canonical`; `test_a16_row_key_not_position`; `test_a16_stale_row_drift_voids` (H18). **Adversarial.** H18. **Open.** per-tenant sheet schemas — NEEDS VALIDATION.
