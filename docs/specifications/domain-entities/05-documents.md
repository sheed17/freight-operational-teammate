# Domain Family 05 ‚Äî Documents *(Document ¬∑ Document Requirement ¬∑ Document Packet)*

*Registry applies. ### **NO new Claim/Evidence primitive** ‚Äî a Document REUSES the foundational Evidence (immutable artifact) + Identity Binding Claim (which entity it belongs to).*

---
## E22 ‚Äî Document ¬∑ lifecycle **L-Doc**
**Def.** A retained freight artifact (POD, BOL, Rate Con, lumper receipt, invoice, COI, scale ticket‚Ä¶) ‚Äî ### **the domain wrapper over a foundational Evidence + its binding.** **Not.** ### **Not the Evidence bytes** (those are immutable, content-addressed); not the extracted values (those are Claims); not the Document Packet. **Authority.** ### **field-level:** the **content** is projected+immutable (Evidence); the **classification** and **binding** are native Claims (`MODEL_EXTRACTED`‚Üíre-matched deterministically). **Identifier.** `document_id`; ### **the Evidence `content_digest` is the strongest natural id (deduplicates).** **Collision.** ### **duplicate documents (same digest) deduplicate; a revised document is a NEW digest ‚áí a new version.** **Enums.** `doc_type ‚àà {POD, BOL, RATE_CON, LUMPER_RECEIPT, SCALE_TICKET, COI, CUSTOMER_INVOICE, CARRIER_INVOICE, ‚Ä¶}` (additive). ### **An UNSIGNED BOL is NOT a POD (the POD gate).** **L-Doc states.** `EXPECTED ‚Üí RECEIVED ‚Üí EXTRACTED ‚Üí BOUND ‚Üí FILED` ¬∑ `ILLEGIBLE` ¬∑ `REJECTED` ¬∑ `SUPERSEDED`. **Key transitions:** `EXPECTED` creates an **Expectation** (#8-family); `BOUND` via deterministic Identity Binding (CD-6); `FILED` is a consequential effect (`FILE_DOCUMENT` Pipeline ‚Üí `EffectVerified`). **Correction.** ### **content immutable; a wrong binding is corrected via ClaimCorrected (propagates, retains history ‚Äî CD-6/CD-7); a re-bind NEVER silently moves the doc (CD-6).** **Hostiles:** #8 (doc references one load# but belongs to another customer) ‚áí the load# is `MODEL_EXTRACTED` evidence, deterministically matched **within `(tenant, customer)`** ‚Äî a cross-customer match is `AMBIGUOUS`/`Conflict`, never auto-bound; #9 (one POD covers two loads) ‚áí **1:N Document‚ÜîLoad binding**, both bound, neither silently; #10 (illegible) ‚áí `ILLEGIBLE` ‚áí Exception; #24 (a binding correction invalidates a completed invoice) ‚áí Compensation. **Fail-closed.** ### **an `ILLEGIBLE`/`AMBIGUOUS`/missing required document blocks billing (CD-3).** **Acceptance.** `test_unsigned_bol_is_not_pod`; `test_document_never_silently_rebinds`; `test_one_pod_two_loads_binds_both`; `test_illegible_blocks_billing`. **Invalid-impossible.** editing document content; a doc moving loads without a `ClaimCorrected`. **Open.** doc-type enum per customer; handwritten-extraction confidence ‚Äî NEEDS VALIDATION (fail-closed to human).

## E23 ‚Äî Document Requirement
**Def.** ### **A policy-derived statement that a given Load/Customer/lane requires document types D before a consequential action.** **Not.** A Document; not a Packet. **Authority.** native (compiled from a **Rule** ‚Äî e.g. *"never bill without a POD"*, or *"Customer Y requires a lumper receipt"*). **Identifier.** `requirement_id`. **Attrs.** `required_doc_types[]`, `scope` (customer/lane/action), `gate` (blocks which action class). **Rule interaction.** ### **a requirement is a compiled `GATE_PRECONDITION` Rule (M12); a natural-language "Customer Y requires X" either compiles or is honest memory (M-52).** ### **Requirements VARY BY CUSTOMER (hostile #11).** **CD-3:** ### **the Customer-required packet gates invoice release; an override needs an authorized human exception.** **Acceptance.** `test_requirement_compiles_or_is_memory`; `test_customer_specific_requirement_gates_invoice`. **Open.** per-customer requirement sets ‚Äî NEEDS VALIDATION.

## E24 ‚Äî Document Packet ¬∑ lifecycle **L-Packet**
**Def.** ### **The set of Documents assembled to satisfy a Load's Document Requirements for a consequential action (billing/settlement).** **Not.** A single Document; not a file. **Authority.** native. **Identifier.** `packet_id`. **L-Packet states.** `ASSEMBLING ‚Üí COMPLETE ‚Üí SUBMITTED` ¬∑ `INCOMPLETE` ¬∑ `SUPERSEDED`. **Key transitions:** `COMPLETE` (guard: ### **every required doc type present, `consistent`, correctly bound, from an acceptable provenance ‚Äî a `MODEL_INFERRED` POD does NOT count**); `INCOMPLETE` blocks billing (CD-3). **Multi-document files / missing pages / conflicting versions:** hostile ‚Äî one file ‚Üí many Documents; a missing page ‚áí `INCOMPLETE`; conflicting versions ‚áí `ConflictRaised`. **Acceptance.** `test_packet_complete_requires_consistent_bound_docs`; `test_incomplete_packet_blocks_billing`. **Invalid-impossible.** a `COMPLETE` packet with a `MODEL_INFERRED` or `AMBIGUOUS`-bound required doc. **Open.** none beyond E23.
