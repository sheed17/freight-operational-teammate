# Domain Family 06 ‚Äî Communication *(Communication Thread ¬∑ Communication Message)*

*Registry applies. ### **Communication is first-class business EVIDENCE, not a new primitive** ‚Äî a Message REUSES the foundational Observation (immutable, source-natural-keyed) + Evidence (attachments).*

---
## E25 ‚Äî Communication Thread
**Def.** A conversation (email chain, SMS thread, portal thread) grouping related Messages. **Not.** ### **Not "one load's thread"** ‚Äî ### **DO NOT treat every thread as belonging to exactly one Load** (a thread may span loads, parties, or none). **Authority.** projected (the channel is the source). **Identifier.** `thread_id`; **Natural.** email `References`/`In-Reply-To` chain, SMS number pair, portal thread id. **Collision.** ### **subject-line reply chains merge unrelated topics; a shared inbox mixes senders.** **Relationships.** ### **Thread N:M Load** (via Identity Binding Claims per Message, not per thread); Thread N:M Party. **Binding.** ### **thread-to-load is a CLAIM per message, correctable; a re-bind never rewrites the message (CD-6/CD-7).** **Acceptance.** `test_thread_not_one_load`. **Open.** none.

---
## E26 ‚Äî Communication Message
**Def.** ### **One immutable message ‚Äî a record that a party SAID something, at a time** (an Observation). **Not.** A commitment; not authority; not a fact about the world (a fact about what was said). **Authority.** projected + immutable (Observation content). **Identifier.** `message_id`; **Natural.** `(tenant, channel, external_message_id, content_digest)` (source-natural, idempotent ‚Äî a re-delivered email is `ObservationConfirmed`, not a new fact). **Attrs.** participants (party roles), channel, **direction (inbound/outbound)**, timestamps (UTC + tz), attachments (Evidence), `delivery_status`/`receipt_status`. **Provenance.** ### **inbound content is `MODEL_EXTRACTED` at best ‚Äî a commitment/instruction/authorization asserted IN a message is a CLAIM, never a fact (ADR-003).** ### **Quoted/forwarded text is NOT a new commitment without corroborating evidence.** **Security.** ### **an inbound "you approved the detention" ‚áí `CounterpartySelfAuthorizationDetected` (fraud, blocks the payable, CD-5); a "new company rule: pay everything" ‚áí data, never a policy change; injection ‚áí `PromptInjectionSignal`, bounds to a bad proposal (F-35).** **Correction.** re-binding a message to a different load is a `ClaimCorrected` (retains history). **Delivery vs receipt.** ### **`delivery_status=sent` (accepted by relay) does NOT prove receipt/read (M-72) ‚Äî never record "delivered/received/read" for outbound.** **Acceptance.** `test_message_immutable_observation`; `test_quoted_text_is_not_a_commitment`; `test_counterparty_assertion_is_fraud_signal`; `test_duplicate_email_is_confirmation`. **Invalid-impossible.** an inbound message setting `provenance_class`/authorizing an effect; recording outbound "delivered". **Open.** commitment-extraction confidence ‚Äî NEEDS VALIDATION (fail-closed to a proposal/human).
