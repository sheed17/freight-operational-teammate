# Machine M9 — Exception

*Registry: `registry.md`. Entity: `entities/12-exception.md`. Lifecycle: Target Spec §12.9; F-30.*

> The brief names finer sub-states (triage / assigned / investigating / awaiting-external / awaiting-human / resolution-proposed). ### **These are `owner`/`sub_status` FIELDS on the row, NOT new lifecycle states** — the canonical state set is the frozen five (`OPEN, ACKNOWLEDGED, AGEING, ESCALATED, RESOLVED`). Adding lifecycle states would diverge from the entity spec; the finer distinctions are captured as an enum `sub_status` under `OPEN`/`ACKNOWLEDGED` without new transitions.

Unknown-outcome.** n/a (an unknown outcome elsewhere *raises* an Exception here).

## 15. Illegal transitions. ### **`RESOLVED` without a valid `decision_ref` → ILLEGAL (GR-14).** ### **`AutoClose` / `Inactivity` → ILLEGAL** (an exception closed without a decision is not closed — it is forgotten). An ownerless Exception → ILLEGAL. An expired Exception → ILLEGAL (never expires). **16. Precedence.** resolution (EC-3/6) requires a human; ageing/escalation are automatic but never resolve. **17. Concurrency.** OCC; optional `UNIQUE(tenant,source_ref,type) WHERE state!='RESOLVED'` dedups. **18. Version-check.** n/a. **19. Idempotency.** GR-4 + the dedup index (a re-raise of the same cause is a no-op). **20. Retry.** n/a (a recurrence is a new Exception). **21. Replay.** GR-11.
## 22. Cancellation. only if the underlying cause is retracted — still an event, still a `decision_ref`. **23. Expiry.** never. **24. Reopening.** a recurrence is a new Exception (a new linked phase on the source work item, GR-12). **25. Correction.** n/a. **26. Supersession.** n/a. **27. Compensation.** n/a directly. **28. Human-approval.** an open Exception that freezes an entity blocks consequential actions on it (via GR-10/the frozen field). **29. Policy.** ### **a repeatedly-overridden rule raises an Exception to the Policy Owner ("overridden 6 times — should it change?") — the system ASKS; it does not change the rule** (GR-15 area). **30. Brake.** Sev-0 exceptions (orphan adapter, cross-tenant breach, rebuild divergence) auto-engage the brake via their source detectors.
## 31. Evidence/provenance. `[C-7]`; a PERMANENT-failure exception records the failure classification. **32. Events.** EC-* per registry. **33. Consumes.** `Acknowledged, Resolved, TimerFired`. **34. Writes.** exception row + (where applicable) entity-freeze + outbox. **35. Txn.** raise+freeze one commit. **36. Crash recovery.** an open Exception persists; the frozen entity stays blocked until resolved. **37. Timeout.** EC-4/EC-5 (ageing/escalation) — durable timers; ### **never a resolution timer.** **38. Observability.** ### **`NEEDS_VERIFICATION`-backed and Sev-0 exceptions are the highest-priority queue; mean-time-to-human-resolution is the metric that matters** (it measures escalation-message quality). **39. Audit.** raise/ack/age/escalate/resolve with actor and `decision_ref`. **40. Security.** ### **a model may NEVER resolve/auto-clear an Exception** (GR-7); resolution requires an authenticated human with a valid `decision_ref`.
## 41. Acceptance. (a) closure without a valid `decision_ref` is illegal; (b) inactivity never closes; (c) an owner exists from creation; (d) a permanent failure raises immediately with zero retries; (e) ageing escalates via a durable timer.
