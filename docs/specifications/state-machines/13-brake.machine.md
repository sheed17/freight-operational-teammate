# Machine M13 — Brake

*Registry: `registry.md`. Entity: `entities/16-brake.md`. Lifecycle: Target Spec §12.13; ADR-011.*

> ### **TWO states — `ACTIVE`, `RELEASED`. Necessity of no more is proved in `entities/16-brake.md` point 4.2:** "engaged by human vs automation" is an `actor` FIELD; "partially released" is a `scope` change; "pending release" would require a release approval workflow, which is forbidden (requiring ceremony to become safer is a design error).

**1.** Brake. **2.** Brake. **3. Purpose.** Admission control — refuse to mint and refuse to claim; ### **NEVER kill a worker** (that would manufacture an `UNKNOWN_OUTCOME`). **4. Aggregate/txn.** ### **engagement is a SINGLE ROW WRITE** (works mid-incident); checkpoint step 7 + the claim CAS read brake state in the same transaction as the other checks. **5. Owner.** the engaging actor (human or detector); release is a human. **6. Initial.** `ACTIVE` (on engage). **7. States.** `ACTIVE, RELEASED`. **8. Terminal.** `RELEASED`. **9. Non-terminal.** `ACTIVE`. **10. Recoverable.** `ACTIVE`. **11. Failure.** n/a. **12. Expiry.** ### **NEVER — no TTL; a brake that expires releases itself while nobody is looking.** **13. Unknown-outcome.** n/a.

## 15. Illegal transitions. ### **automation releasing or narrowing a brake → ILLEGAL (GR-16 / one-way ratchet).** A model engaging/releasing → ILLEGAL. A timer releasing → ILLEGAL. Release without a `decision_ref` → ILLEGAL. Release that reactivates stale witnesses/grants → prevented by `brake_version` (BR-4 bumps it, invalidating all). A third state → ILLEGAL. **16. Precedence.** the brake sits at policy-precedence layer 3 — ### **it denies regardless of what policy permits.** **17. Concurrency.** `UNIQUE(tenant,scope) WHERE state='ACTIVE'`; ### **`brake_version` monotonic per scope-owner; the platform `GLOBAL` brake is ONE row (SD-12); witnesses/grants bind BOTH `global_brake_version` and `tenant_brake_version`, both re-validated in the claim CAS.** **18. Version-check.** the claim CAS re-validates both brake versions — ### **a brake engaged between mint and claim makes the CAS match zero rows (never both, never neither).** **19. Idempotency.** ### **engagement is idempotent on scope — a flapping detector is one `ACTIVE` brake + a rising signal count; it cannot self-release, so flapping opens no window.** **20. Retry.** n/a. **21. Replay.** GR-11 — brake history replays; it never re-engages a real brake.
## 22. Cancellation. release (BR-4) is the only exit. **23. Expiry.** never. **24. Reopening.** a new incident is a new brake. **25. Correction.** n/a. **26. Supersession.** n/a (scope changes are BR-2/BR-3). **27. Compensation.** ### **compensation is BLOCKED under an active brake** (a misbehaving system must not write corrections). **28. Human-approval.** ### pending approvals remain RECORDED but cannot authorize execution; `BrakeEngaged` ⇒ M4 `VOID_ON_BRAKE`. **29. Policy.** orthogonal; denies regardless. **30. Brake.** (self).
## 31. Evidence/provenance. `actor` records who/which detector; release requires positive integration-health evidence (a positive control, not "the page loaded"). **32. Events.** BR-* per registry. **33. Consumes.** Sev-0 detector signals; human engage/release commands. **34. Writes.** the single brake row + outbox. **35. Txn.** engagement one row write; checkpoint/claim read it transactionally. **36. Crash recovery.** the brake row persists; ### **the brake store unreachable at checkpoint ⇒ no effect possible ("cannot read the brake" NEVER means "off").** **37. Timeout.** none (BR-5 illegal). **38. Observability.** ### **a hidden brake violates R17 — every surface reports it unprompted: scope, what is STILL allowed, reason, actor (human or named detector), time, prevented effects, in-flight effects + status, unresolved unknown outcomes + exposure, exact release requirements.** **39. Audit.** engage/widen/narrow/release with actor, reason, `brake_version`, `decision_ref`; the brake report is the incident timeline. **40. Security.** ### **automation may engage/widen; NEVER release/narrow; a detector may never clear its own alarm.** In-flight effects at positions 3/4/5 (claimed/executing/verifying) RUN TO A VERIFIED CONCLUSION — the brake stops the NEXT effect, not the LAST.
## 41. Acceptance. (a) engaging during an adapter call does not create an unknown outcome; (b) brake-between-mint-and-claim ⇒ CAS zero rows (10,000× interleaved: never both, never neither); (c) engages with the policy engine + TMS down; (d) automation engages but never releases; (e) release re-checkpoints all queued work; (f) never auto-expires; (g) unresolved unknown outcomes don't block release but stay frozen + owned.
## 42. Adversarial. Traces 2 (brake between checkpoint and claim), 17 (release while unknowns remain).
