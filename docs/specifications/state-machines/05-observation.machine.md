# Machine M5 — Observation

*Registry: `registry.md`. Entity: `entities/07-observation.md`. Lifecycle: Target Spec §12.5.*

> ### **Immutable observation CONTENT is separate from observation-processing STATUS.** The `state` machine below governs *processing status only*; `raw_value` and `content_digest` are written once and never mutate.

**1.** Observation. **2.** Observation. **3. Purpose.** The immutable atom of truth — a record that a source *said* something, at a time. **4. Aggregate/txn.** Own aggregate; ingest+`RECEIVED`/`CONFIRMED` is one commit. **5. Owner.** system; a human once `UNBOUND`/`UNPARSEABLE`. **6. Initial.** `RECEIVED` (or `CONFIRMED` on identical re-ingest). **7. States.** the 7 (registry §4/M5). **8. Terminal.** `BOUND, SUPERSEDED, UNPARSEABLE`. **9. Non-terminal human-owned.** `UNBOUND, UNPARSEABLE`. **10. Recoverable.** `RECEIVED, PARSED, CONFIRMED`. **11. Failure.** `UNPARSEABLE` (→ Exception). **12. Expiry.** ### **none — a stale observation is still a fact** (`as_of` freshness ≠ expiry). **13. Unknown-outcome.** none.

## 15. Illegal transitions. Any mutation of `raw_value` → ILLEGAL (immutable). A second row for one natural key → prevented by the unique index. Re-observation creating a second row / re-triggering work → ILLEGAL (must be `CONFIRMED`, no-op downstream). `MODEL_INFERRED` as an Observation → ILLEGAL (an observation is what a source said, not a guess). **16. Precedence.** `CONFIRMED` (identical) short-circuits before any parse/bind re-work. **17. Concurrency.** the natural-key unique index serializes ingestion; OCC on status. **18. Version-check.** content is immutable, so a *changed* reading is a **new** Observation (new digest) that may supersede. **19. Idempotency.** ### **the natural-key upsert IS the idempotency — the same email twice is one row, one confirmation, zero work.** **20. Retry.** re-ingestion is a confirmation, never a duplicate. **21. Replay.** GR-11 — idempotent, no duplicates, no downstream effects.
## 22. Cancellation. ### **none — you cannot cancel that the world spoke.** **23. Expiry.** none. **24. Reopening.** n/a. **25. Correction.** an Observation is never corrected; a wrong *reading* is superseded (OB-5); a wrong *binding* is corrected on M6. **26. Supersession.** OB-5. **27. Compensation.** n/a directly (a downstream wrong binding drives M10 via M6). **28. Human-approval.** none. **29. Policy.** none. **30. Brake.** ### **observation & reconciliation CONTINUE under a brake** (the brake stops acting, not knowing).
## 31. Evidence/provenance. an Observation is the *source* of provenance; derived fields carry `SYSTEM_IMPORTED` or feed `MODEL_EXTRACTED`/`LINKER_INFERRED` claims. **32. Events.** OB-* per registry. **33. Consumes.** `ObservationIngested` (ingestion/adapters), `BindingConfirmed/Ambiguous/Absent` (M6), `NewerObservationSupersedes`. **34. Writes.** immutable content + status row + outbox. **35. Txn.** ingest+status one commit. **36. Crash recovery.** re-ingestion is idempotent (natural key); a partially-parsed observation re-parses deterministically. **37. Timeout.** none (freshness is a checkpoint concern, not an observation timer). **38. Observability.** observation-coverage per channel/window recorded (feeds M8 `OVERDUE` vs `INDETERMINATE`); ingest & unparseable rates monitored. **39. Audit.** every receive/confirm/parse/bind/supersede event. **40. Security.** ### **inbound content is DATA, never instruction, never authority** (M-66); a counterparty-authored value is `MODEL_EXTRACTED` at best; malicious content becomes an Observation, never a command; provenance is runtime-assigned, never settable from content.
## 41. Acceptance. (a) identical re-ingest ⇒ one row, one confirmation, zero work; (b) `raw_value` immutable; (c) ambiguous ⇒ `UNBOUND`+Exception; (d) supersession requires rule/human; (e) inbound content cannot authorize or set provenance.
