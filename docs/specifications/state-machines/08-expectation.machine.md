# Machine M8 — Expectation

*Registry: `registry.md`. Entity: `entities/11-expectation.md`. Lifecycle: Target Spec §12.8; F-14/F-25.*

**1.** Expectation. **2.** Expectation. **3. Purpose.** Time-driven and *non-event* work; distinguish ### **`OVERDUE` ("it never came") from `INDETERMINATE` ("we weren't watching")** (I8). **4. Aggregate/txn.** Own aggregate; the deadline `TimerFired` + coverage read + resulting state is one commit. **5. Owner.** system; a human on `OVERDUE`/`INDETERMINATE`. **6. Initial.** `RAISED`. **7. States.** the 6 (registry §4/M8). **8. Terminal.** `DISCHARGED, CANCELLED, EXPIRED`. **9. Non-terminal human-owned.** `OVERDUE, INDETERMINATE`. **10. Recoverable.** `RAISED`. **11. Failure.** `OVERDUE`/`EXPIRED` (→ Exception). **12. Expiry.** `EXPIRED` (terminal age past overdue/indeterminate → Exception). **13. Unknown-outcome.** ### **`INDETERMINATE` IS the observability-unknown** (distinct from `OVERDUE`).

## 15. Illegal transitions. ### **`OVERDUE` without a healthy-coverage `coverage_ref` → ILLEGAL** (must be `INDETERMINATE`). A deadline evaluated in the wrong timezone → ILLEGAL (facility-local, F-25). Two `RAISED` for one `expectation_key` → prevented by `UNIQUE(...) WHERE state='RAISED'`. Silent expiry (no Exception) → ILLEGAL. **16. Precedence.** discharge (EX-2/EX-4) beats overdue/indeterminate — a late arrival always discharges. **17. Concurrency.** the `expectation_key` unique index + OCC. **18. Version-check.** n/a. **19. Idempotency.** GR-4 + the key index. **20. Retry.** n/a. **21. Replay.** GR-11 — coverage-based `OVERDUE`/`INDETERMINATE` decisions replay deterministically from the coverage records.
## 22. Cancellation. EX-6. **23. Expiry.** EX-7 (→ Exception, never silence). **24. Reopening.** n/a. **25. Correction.** a wrong expectation is `CANCELLED`, not corrected. **26. Supersession.** a re-versioned deadline (EX-5) is not a supersession. **27. Compensation.** n/a. **28. Human-approval.** none. **29. Policy.** ### **a compiled rule like "Customer Y requires hourly updates" IS an Expectation, not a gate** (it owes, it does not authorize). **30. Brake.** discharge & `INDETERMINATE` detection continue under a brake (observation continues).
## 31. Evidence/provenance. discharge requires a bound Observation; `INDETERMINATE` asserted from a negative/absent coverage record. **32. Events.** EX-* per registry. **33. Consumes.** `ObservationBound, TimerFired, DeadlineChanged, ReasonDisappeared`. **34. Writes.** expectation row + coverage read + outbox. **35. Txn.** the deadline transition is one commit. **36. Crash recovery.** the durable timer re-fires; the coverage record determines `OVERDUE` vs `INDETERMINATE` deterministically. **37. Timeout.** EX-3/EX-3i (deadline) + EX-7 (terminal age) — durable timers, not sweeps. **38. Observability.** the `OVERDUE`-vs-`INDETERMINATE` split is a monitored honesty metric; ### **facility/appointment windows evaluated in the FACILITY's local timezone across DST.** **39. Audit.** every raise/discharge/overdue/indeterminate/cancel/expire with the coverage basis. **40. Security.** a model may propose an Expectation; deadline+coverage are runtime-set.
## 41. Acceptance. (a) deadline passes while blind ⇒ `INDETERMINATE`; (b) late arrival discharges; (c) duplicates prevented; (d) appointment window evaluates facility-local across DST; (e) expiry raises an Exception.
