# Machine M12 — Rule

*Registry: `registry.md`. Entity: `entities/15-rule.md`. Lifecycle: Target Spec §12.12; ADR-010 §6.*

> The brief names "parsed / invalid / conflict-detected / awaiting-confirmation / suspended". ### **Mapped to the frozen set without new states:** *parsed* = a `PROPOSED` candidate; *invalid* = `REJECTED` (`RuleNotEnforceable`); *conflict-detected* = a raised **M7 Conflict** (the rule stays `COMPILED`, blocked); *awaiting-confirmation* = `COMPILED`; *suspended* = `REVOKED`.

**1.** Rule. **2.** Rule. **3. Purpose.** Resolve L-C: a human instruction either **compiles into an enforceable Rule** or **remains non-authoritative memory and the owner is told it is not a rule** — no third state implies enforcement without compilation. **4. Aggregate/txn.** Own aggregate; compilation is deterministic (no model after the text proposal). **5. Owner.** the Policy Owner. **6. Initial.** `PROPOSED`. **7. States.** the 8 (registry §4/M12). **8. Terminal.** `REJECTED, SUPERSEDED, REVOKED, EXPIRED`. **9. Non-terminal.** `PROPOSED, COMPILED, CONFIRMED, ACTIVE`. **10. Recoverable.** all non-terminal. **11. Failure.** `REJECTED` (`RuleNotEnforceable`) — ### **and the instruction becomes organizational memory, the owner TOLD it is not a rule.** **12. Expiry.** `EXPIRED` (narrowing only, human at expiry). **13. Unknown-outcome.** none.

## 15. Illegal transitions. ### **a reply claiming enforcement with no `ACTIVE` rule_id → ILLEGAL / defect (the L-C guard, asserted on literal reply text).** A model activating a rule → ILLEGAL. A rule branching on `MODEL_INFERRED` → never compiles (RU-2f). Auto-merged conflicting rules → ILLEGAL (RU-3 fails closed). **16. Precedence.** rules sit at policy-precedence layer 6; ### **the narrower scope wins; genuine conflicts fail closed.** **17. Concurrency.** `UNIQUE(tenant,scope,kind) WHERE state='ACTIVE'` where a scope admits one; OCC. **18. Version-check.** `rule_version` monotonic; a decision is explained under the rule versions at its checkpoint. **19. Idempotency.** GR-4; re-activating an active version is a no-op. **20. Retry.** n/a. **21. Replay.** GR-11.
## 22. Cancellation. `REVOKED` (RU-7). **23. Expiry.** RU-8. **24. Reopening.** n/a. **25. Correction.** superseded (retained). **26. Supersession.** RU-6. **27. Compensation.** n/a. **28. Human-approval.** RU-4/RU-5. **29. Policy.** a `GATE_PRECONDITION`/`CONSTRAINT` rule is evaluated in checkpoint step 6 (e.g. "never bill without a POD" ⇒ `pod.evidence_condition=consistent` AND `pod.provenance_class∈{SYSTEM_IMPORTED,OWNER_ASSERTED,MODEL_EXTRACTED-with-artifact}`; a `MODEL_INFERRED` POD ⇒ DENY). **30. Brake.** rule evaluation is part of the checkpoint the brake gates.

## 41. Acceptance. (a) "do not use Carrier X for produce" cannot compile (commodity unmodelled), owner told; (b) "never bill without a POD" compiles to a precondition; (c) a reply never claims enforcement without an active rule_id; (d) two conflicting rules fail closed; (e) a model cannot activate.
