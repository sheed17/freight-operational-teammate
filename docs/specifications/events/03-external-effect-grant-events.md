# Event Family F3 — External Effect / Effect Grant Events

*Registry: `events/registry.md`. Producer: machine M3 (the one `effect_grants` row). Envelope + defaults: registry §1–§2.*

**Family defaults:** `aggregate_type=effect_grant`; ordering = ### **STRICT per-aggregate** (the eight-state row is version-monotonic); security=`high` (these are the money events); projection-permitted = **none directly** (`ProjectionUpdated` is F2); `work_item_id`+`pipeline_instance_id`+`commit_key` on all; **consequential** ⇒ pin the SD-3 set + fingerprint + policy/brake versions.

## Cross-cutting
**Dedup:** transition-natural; the claim CAS makes `GrantClaimed`/`EffectAttempted` exactly-once. **Ordering:** STRICT per-grant (`EffectVerified` before `GrantClaimed` is impossible; parked if misdelivered). **Replay:** ### **replaying ANY F3 event mints zero grants, claims zero, produces zero `EffectAttempted`, touches the world zero times** (ER-2) — this is the family where GR-11 matters most. **Projection:** none directly; a verified effect drives F2 `ProjectionUpdated`. **Security:** `EffectAttempted`-without-a-claimed-grant, double-claim, and stale-witness all map to F14 with auto-brake per registry §11. **Audit:** `high` — the grant + witness + fingerprint are the "why did you pay" record (§11). **Open validation:** grant TTL; V7 (commit_key in the external record for deterministic `RealityEstablished`) — fail-closed (a human resolves otherwise).
