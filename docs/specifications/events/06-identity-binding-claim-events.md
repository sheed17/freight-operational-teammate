# Event Family F6 — Identity Binding Claim Events

*Registry: `events/registry.md`. Producer: machine M6. Defaults: registry §1–§2.*

**Family defaults:** `aggregate_type=identity_binding_claim`; ordering = **order-tolerant except correction/supersession** (per-aggregate version guards those); security=`internal`; projection = ### **claim events write NATIVE binding state; they NEVER overwrite `OWNER_ASSERTED` and never write projected truth optimistically (ER-14, §7)**; every event carries `provenance_class` + `match_method`.

|---|---|---|---|---|

## Cross-cutting
**Dedup:** transition-natural; re-proposing an identical `LINKER_INFERRED` binding for a `CONFIRMED` subject is a no-op. **Ordering:** correction/supersession STRICT (a `ClaimCorrected` before its `ClaimConfirmed` is **parked**, T13); proposals order-tolerant. **Replay:** ### **every `OWNER_ASSERTED` binding replays byte-identical (a rebuild rebuilds projections, not the owner's mind, ER-2)**; correction-of-correction re-runs propagation deterministically. **Projection:** claim events write native binding state and may feed a projected canonical binding when deterministic; ### **a claim event NEVER silently overwrites `OWNER_ASSERTED` (ER-14); an inferrer disagreement raises `ConflictRaised` (F7), not an overwrite.** **Security:** ### **a machine-actor `ClaimConfirmed{OWNER_ASSERTED}` is illegal ⇒ `OwnerAssertedOverwriteAttempted` (F14)**; a payload attempting to strengthen provenance ⇒ `ProvenanceStrengtheningAttempted`. **Open validation:** V4; migration hazard — existing owner corrections must be re-captured as `OWNER_ASSERTED`.
